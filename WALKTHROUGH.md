# LeadSpark v2 — UX Walkthrough (ฉบับปรับใหม่)

> **เป้าหมายเอกสารนี้:** อธิบาย user journey ทุกหน้าหลังการ refactor UX
> Developer อ่านแล้วต้องเข้าใจว่า user จะเห็นอะไร กดอะไร เกิดอะไรขึ้น โดยไม่ต้องเปิดแอป

---

## แนวคิดหลัก

**LeadSpark** = B2B Marketplace + Smart Work Hub สำหรับ SME ไทย
ส่วนนี้เป็น walkthrough ของฝั่ง **"จัดการขาย" (Work Hub)**

**Reference platforms:**
- Pipedrive → pipeline UX
- HubSpot → CRM patterns
- Todoist → task completion flow
- Linear → panel UX, keyboard shortcuts
- Freshbooks → document creation wizard

**Tech context:** Next.js 14, Zustand store, shadcn/ui, Tailwind CSS
Mock data ทั้งหมด — ไม่มี API จริง, ข้อมูลหาย reload

## Global UX Patterns (ใช้ทุกหน้า)

### Pattern 1 — SlideOverPanel
> **"กดชื่ออะไร → ดูข้อมูลทางขวา ไม่ต้องออกจากหน้า"**

```
ทุกที่ที่กดชื่อ item (task, customer, lead, document):
→ Panel เลื่อนเข้าจากขวา (40-50% ของจอ)
→ Backdrop สีดำจาง ซ้ายยังเห็น list เดิม
→ กด Escape หรือ backdrop → panel ปิด
→ ไม่ redirect ออกจากหน้าปัจจุบัน

Panel anatomy:
┌─────────────────────────────────┐
│ [ชื่อ item]   [status]    [✕]   │  ← Header (sticky)
├─────────────────────────────────┤
│                                 │
│  [ข้อมูล detail + timeline]     │  ← Body (scrollable)
│                                 │
├─────────────────────────────────┤
│  [Action 1]        [Action 2]   │  ← Footer (sticky)
└─────────────────────────────────┘
```

---

### Pattern 2 — CheckboxActionBar
> **"ติ๊กแล้วไม่หายทันที — ต้องยืนยันก่อน"**

```
สถานะ flow:

[☐] Task name  →  ติ๊ก  →  [☑] ̶T̶a̶s̶k̶ ̶n̶a̶m̶e̶  (opacity 50%)
                                 ↓
                          [✓ ยืนยัน] [↩ ยกเลิก] [📝 หมายเหตุ]
                          [████████░░] countdown 5 วิ
                                 ↓
                     ยืนยัน หรือ รอ 5 วิ
                                 ↓
                     Task เลื่อนไป "เสร็จแล้ว" (animated)
```

กด "ยกเลิก" ตอนไหนก็ได้ → คืนสภาพปกติ
กด "หมายเหตุ" → พิมพ์ note → Enter → ยืนยัน + บันทึก note

---

### Pattern 3 — SmartFilterBar
> **"Filter เป็น chip ที่ลบได้ทีละอัน"**

```
[🔍 ค้นหาชื่อธุรกิจ...] [สถานะ ▾] [อุตสาหกรรม ▾] [+ เพิ่มตัวกรอง]

Active filters (ถ้ามี):
[สถานะ: active ✕] [อุตสาหกรรม: ร้านอาหาร ✕] [ล้างทั้งหมด]
```

---

### Pattern 4 — EmptyState
> **"List ว่าง → บอก user ว่าต้องทำอะไร"**

```
        [📋 icon]
    ยังไม่มีงานวันนี้
  งานที่ครบกำหนดวันนี้จะแสดงที่นี่
  [+ เพิ่มงาน]   [📥 นำเข้า]
```

## หน้าหลัก (Dashboard) — `/`

### เมื่อ User เปิดแอปครั้งแรกของวัน:

**ส่วน 1 — Morning Brief (ด้านบนสุด)**
```
สวัสดีตอนเช้า Asia 👋
วันนี้มี [3 งาน] · [2 ลูกค้าต้อง follow up] · ยอดขายเดือนนี้ [68%] ของเป้า
```
- `[3 งาน]` กดได้ → scroll ลงไปที่ TodayTasks section
- `[2 ลูกค้า]` กดได้ → scroll ลงไปที่ CustomerAttention section
- `[68%]` กดได้ → navigate ไป `/reports`
